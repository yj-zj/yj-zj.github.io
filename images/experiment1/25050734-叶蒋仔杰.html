实验一 网络软件基础

一、实验目的

  1. 安装并运行一个桌面网络软件（Cisco Packet Tracer）；
  2. 理解C/S、B/S、P2P等不同网络软件类型与架构；
  3. 通过动手搭建一个简单的局域网，直观感受网络软件通信的基本原理。

二、实验环境

  • 操作系统：Windows 10/11
  • 网络：需要互联网连接以下载软件
  • 软件：Cisco Packet Tracer（免费，需注册Cisco NetAcad账号）

三、实验过程

步骤一：注册Cisco NetAcad账号并下载Packet Tracer

  1. 打开浏览器访问Cisco Networking Academy官网，点击页面右上角注册按钮。
  2. 依次填写所在地区、出生年月、邮箱等信息，完成账号注册并邮箱验证。
  3. 登录账号后进入学习资源页面，找到Cisco Packet Tracer下载入口，选择Windows 64位版本安装包下载。

步骤二：安装Cisco Packet Tracer

  1. 找到下载完成的PacketTracer，右键选择以管理员身份运行启动安装向导。
  2. 点击Install开始安装，等待进度条走完后点击Finish完成安装。

步骤三：搭建局域网硬件拓扑

  1. 在左侧设备栏选择Switches，拖拽2960交换机至工作区；再进入End Devices拖拽两台PC（PC0、PC1）。
  2. 点击连线工具，选择Copper Straight-Through直通线。
  3. PC0 FastEthernet0端口连接交换机FastEthernet0/1；PC1 FastEthernet0端口连接交换机FastEthernet0/2。

步骤四：配置两台PC静态IP地址

  1. 双击PC0，切换至Desktop→IP Configuration，选择Static静态配置：
    • IPv4 Address：192.168.1.1
    • Subnet Mask：255.255.255.0
  2. 双击PC1，同样进入IP配置页面：
    • IPv4 Address：192.168.1.2
    • Subnet Mask：255.255.255.0

步骤五：命令行ping连通性测试

  1. 打开PC0的Desktop→Command Prompt命令行窗口。
  2. 输入命令ping 192.168.1.2执行连通测试。
  3. 观察返回数据包回复信息，确认无丢包。

步骤六：仿真模式观察数据包传输流程

  1. 右下角切换至Simulation仿真模式，点击信封图标Add Simple PDU。
  2. 先点击PC0作为源设备，再点击PC1作为目标设备。
  3. 点击自动捕获播放按钮，直观查看数据包从PC0→交换机→PC1完整转发流程，点开数据包可查看各层封装信息。

四、实验结果

4.1 连通性测试汇总表

测试项 源设备(IP) 目标设备(IP) 测试结果 丢包率
ping连通测试 PC0(192.168.1.1) PC1(192.168.1.2) ✅ 成功 0%

4.2 成果说明

  1. 成功完成Cisco Packet Tracer安装、账号登录，软件功能可正常使用；
  2. 完成基础局域网拓扑搭建，设备线缆连接无报错；
  3. 两台终端IP配置在同一网段，双向ping测试全部收到回复，网络通信正常；
  4. 仿真模式下可完整观测数据包转发过程，理解二层交换机数据转发逻辑。

五、知识总结

三种主流网络软件架构对比

架构类型 全称 核心特征 典型例子
C/S架构 客户端/服务器 需要安装专用客户端，两端分工协作 网络游戏、微信、Packet Tracer
B/S架构 浏览器/服务器 仅浏览器即可访问，业务逻辑在服务端 淘宝、知乎、在线文档
P2P架构 点对点 每个节点同时充当客户端与服务器 BT下载、局域网传输工具

5.1 静态站点 vs 动态站点对比

对比维度 静态站点（Hexo/Hugo） 动态站点（WordPress）
页面生成时机 本地提前预编译HTML 用户访问时后端实时渲染
数据库 无需数据库 必须搭配数据库
访问速度 极快，纯静态文件 较慢，需执行后端查询
安全风险 无后端攻击面，安全性高 存在后台、数据库漏洞风险
部署方式 仅上传静态文件 搭建服务器+数据库环境
适用场景 个人博客、技术文档站 电商、论坛、后台管理系统

五、知识总结

5.1 三种虚拟机网络模式对比

网络模式 访问权限 适用场景
NAT 虚拟机可上外网，外部无法主动访问虚拟机 日常软件下载、系统更新
桥接模式 虚拟机与宿主机同局域网，外部设备可访问 需要外部电脑直连虚拟机
仅主机Host-Only 仅宿主机与虚拟机互通，无外网 搭建隔离内网集群实验

5.2 核心操作命令

  1. 关机：sudo shutdown now
  2. 修改主机名：hostnamectl set-hostname xxx
  3. 应用网络配置：sudo netplan apply
  4. 连通测试：ping -c 3 目标地址

六、出现问题

  1. 问题1：虚拟机重启后，ip也会重置
    解决：使用比c50-cloud-init更高优先级的文件来确保静态ip
  2. 问题2：克隆后网卡失效
    解决：在 VirtualBox 虚拟机设置中重新生成网卡 MAC 地址

5.1 Linux 数字权限对照表

数字 权限组合 含义
7 rwx 读、写、执行全部权限
6 rw- 可读、可写，无执行权限
5 r-x 可读、可执行，无写入权限
4 r– 仅可读
0 — 无任何访问权限

配置文件 生效范围 加载时机
~/.bashrc 当前登录用户 每次新开终端窗口
~/.profile 当前登录用户 用户登录时执行一次
/etc/profile 系统全部用户 任意用户登录加载
/etc/profile.d/*.sh 系统全部用户 登录统一加载（推荐）

五、知识总结

5.1 SSH 两种认证方式对比

认证方式 原理 安全等级 适用场景
密码认证 登录输入账号密码验证 较低，易暴力破解 临时测试、内网临时访问
密钥认证 公私钥配对校验身份 高，无密码传输 日常运维、自动化脚本

5.2 SCP 与 rsync 工具区别

工具 特点 适用场景
SCP 完整覆盖传输，无增量判断 少量文件一次性传输
rsync 对比文件差异，仅同步修改内容，支持删除冗余 大型目录定时备份、服务器同步

六、出现问题

  1. 问题1：rsync 远程同步报错找不到命令
    解决：远程服务器安装 rsync：sudo apt install rsync -y
  2. 问题2：ssh 连接提示 Connection refused
    解决：远程服务器未安装 / 未启动 openssh-server，执行sudo systemctl start ssh启动服务

SSH 是开源安全协议，全球服务器运维标准工具，国产服务器操作系统麒麟、统信均内置 OpenSSH 工具，摆脱国外闭源远程管理软件依赖。远程运维技术是云计算、后端运维核心能力，国内云平台大量采用 SSH 管理云主机。
