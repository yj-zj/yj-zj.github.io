实验二 Web软件开发环境

Posted on 2026-07-09 Edited on 2026-07-12 In 实验报告

一、实验目的

  1. 安装并运行一个Web博客软件（本实验选用 Hexo 静态博客框架）；
  2. 了解和掌握Web开发环境的基础安装与配置（Node.js、npm、Git）；
  3. 理解Web软件的B/S架构特点以及静态站点与动态站点的区别。

二、实验环境

  • 操作系统：Windows 10/11
  • 网络：需要互联网连接以下载软件和安装依赖包
  • 所需软件：Node.js（LTS版本，含npm）、Git for Windows、Hexo CLI、VS Code

三、实验过程

步骤一：安装Node.js并验证环境

  1. 访问Node.js官网下载LTS长期支持版本Windows安装包，双击启动安装程序。
  2. 在安装界面勾选「Automatically install the necessary tools」与「Add to PATH」，其余选项保持默认，完成安装。
  3. 打开CMD命令提示符，输入验证命令查看版本。

步骤二：配置npm国内镜像源

步骤三：安装Git

步骤四：全局安装Hexo命令行工具

  1. 在CMD中执行全局安装指令。
  2. 安装结束后输入校验命令查看Hexo版本与依赖信息。

步骤五：初始化Hexo博客项目

  1. 创建存放博客的文件夹并切换目录。
  2. 执行初始化命令自动生成博客项目模板。
  3. 进入项目目录，手动安装项目依赖包。

步骤六：本地启动博客预览服务

  1. 在项目根目录运行本地服务命令：hexo serve
  2. 控制台输出Hexo is running at http://localhost:4000/代表启动成功。

步骤七：新建博客文章并生成静态页面

  1. 使用hexo new命令创建Markdown文章。
  2. VS Code打开生成的md文件，修改文章标题、标签与正文内容并保存。
  3. 执行hexo generate命令编译静态网页。
  4. 重启本地服务，刷新页面查看新增文章。

步骤八：安装并更换NexT主题

  1. 在项目目录下通过Git克隆NexT主题至themes文件夹。
  2. 打开项目根目录_config.yml，修改 heme配置项为
    ext。
  3. 清理缓存、重新生成页面并启动服务预览新主题。

四、实验结果

成果说明

  1. 完整搭建Web开发基础环境，Node.js、npm、Git、Hexo全部安装配置成功；
  2. 成功初始化Hexo静态博客项目，熟悉项目目录各配置文件与文件夹作用；
  3. 实现文章新建、编辑、静态页面生成、本地实时预览全流程；
  4. 完成第三方主题NexT下载与全局配置，博客页面样式成功切换；
  5. 直观验证B/S架构特点，仅通过浏览器即可访问本地Web服务。

五、知识总结

静态博客与动态博客对比

类型 实现原理 优势 典型工具
静态博客 本地预编译Markdown为HTML静态文件 速度快、无数据库、安全性高 Hexo、Hugo、VuePress
动态博客 用户访问时后端实时渲染页面 后台管理、功能丰富、支持评论 WordPress、Typecho

Web开发环境核心组件

组件 核心作用 本次实验工具
运行时环境 解析执行JavaScript代码 Node.js
包管理器 管理项目第三方依赖 npm
版本控制工具 代码备份、多人协作管理 Git
静态站点构建工具 将Markdown转为网页 Hexo CLI

关键命令整理

  1. npm镜像切换：
    pm config set registry https://registry.npmmirror.com
  2. Hexo项目初始化：hexo init 项目名
  3. 新建文章：hexo new “文章标题”
  4. 生成静态页面：hexo generate / hexo g
  5. 本地预览：hexo server / hexo s

六、出现问题

问题：修改_config.yml主题配置后页面无变化
解决：执行hexo clean清理缓存，再重新生成页面并启动服务。

七、心得体会

技术层面

从零搭建了完整的前端Web开发基础环境，掌握Node.js、npm、Git三件套的安装与配置流程，体验了Hexo静态博客框架完成站点搭建、文章创作、主题美化。

思政层面

Hexo、Git、Node.js均为开源工具，依托开源社区免费提供给开发者使用。我们未来要积极参与国内开源项目，完善本土开发工具生态。
