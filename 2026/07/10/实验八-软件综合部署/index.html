实验八 软件综合部署

Posted on 2026-07-10 Edited on 2026-07-12 In 实验报告

一、实验目的

  1. 综合运用前七个实验所学的各类工具；
  2. 构建和模拟一个完整的网络环境；
  3. 完整体验Web静态站点从开发到部署的全过程；
  4. 掌握自动化部署（CI/CD）的基本概念与实践方法。

二、实验环境

  • 操作系统：Windows 10/11 + Ubuntu Server 22.04（虚拟机）
  • 前置条件：已完成实验二、三、四、五、七
  • 所需账号：GitHub账号

三、实验过程

第一部分：GitHub Pages + Actions自动部署

步骤一：本地Hexo项目校验

步骤二：关联远程仓库并推送

步骤三：配置GitHub Actions工作流

步骤四：访问线上站点

第二部分：Linux Nginx手动部署

步骤五：安装Nginx

步骤六：配置Nginx站点

步骤七：SCP传输静态文件

四、实验结果

  1. 完成两套Web站点部署方案（云端自动部署 + 本地Linux服务器部署）；
  2. 掌握CI/CD自动化部署思想，代码提交后自动构建上线；
  3. 熟练使用Git管理项目源码，区分源码仓库与静态部署仓库；
  4. 学会Linux Nginx静态网站部署流程。

五、知识总结

CI与CD概念区分

概念 全称 实验中体现
CI持续集成 Continuous Integration 推送代码自动安装依赖、生成页面
CD持续部署 Continuous Deployment 构建完成自动推送至Pages线上环境

两种部署方案对比

部署方式 优点 缺点 适用场景
GitHub Pages 免费、无需服务器、自动CI/CD 仅静态站点 个人博客、文档站
Linux+Nginx 可控性强、可扩展后端 需维护服务器 企业站点、带后端项目

六、出现问题

rsync远程部署失败（兼容问题）→ 采用scp进行传输

七、心得体会

本次实验是整套软件开发工具实践综合收尾实验，整合了虚拟机、Linux命令、SSH远程、Git版本控制、Hexo静态网站、自动化部署全部知识点。GitHub、Hexo、Nginx、Linux全部为开源免费工具，开源社区降低了软件开发与部署门槛。
