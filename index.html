实验一 网络软件基础

Posted on 2026-07-09 Edited on 2026-07-12 In 实验报告

一、实验目的

  1. 安装并运行一个桌面网络软件（Cisco Packet Tracer）；
  2. 理解C/S、B/S、P2P等不同网络软件类型与架构；
  3. 通过动手搭建一个简单的局域网，直观感受网络软件通信的基本原理。

二、实验环境

  • 操作系统：Windows 10/11
  • 网络：需要互联网连接以下载软件
  • 软件：Cisco Packet Tracer（免费，需注册Cisco NetAcad账号）

三、实验过程

步骤一：注册Cisco NetAcad账号并下载Packet Tracer

  1. 打开浏览器访问Cisco Networking Academy官网，点击页面右上角注册按钮。
  2. 依次填写所在地区、出生年月、邮箱等信息，完成账号注册并邮箱验证。
  3. 登录账号后进入学习资源页面，找到Cisco Packet Tracer下载入口，选择Windows 64位版本安装包下载。

步骤二：安装Cisco Packet Tracer

  1. 找到下载完成的PacketTracer，右键选择以管理员身份运行启动安装向导。
  2. 点击Install开始安装，等待进度条走完后点击Finish完成安装。

步骤三：搭建局域网硬件拓扑

  1. 在左侧设备栏选择Switches，拖拽2960交换机至工作区；再进入End Devices拖拽两台PC（PC0、PC1）。
  2. 点击连线工具，选择Copper Straight-Through直通线。
  3. PC0 FastEthernet0端口连接交换机FastEthernet0/1；PC1 FastEthernet0端口连接交换机FastEthernet0/2。

步骤四：配置两台PC静态IP地址

  1. 双击PC0，切换至Desktop→IP Configuration，选择Static静态配置：
    • IPv4 Address：192.168.1.1
    • Subnet Mask：255.255.255.0
  2. 双击PC1，同样进入IP配置页面：
    • IPv4 Address：192.168.1.2
    • Subnet Mask：255.255.255.0

步骤五：命令行ping连通性测试

  1. 打开PC0的Desktop→Command Prompt命令行窗口。
  2. 输入命令ping 192.168.1.2执行连通测试。
  3. 观察返回数据包回复信息，确认无丢包。

步骤六：仿真模式观察数据包传输流程

  1. 右下角切换至Simulation仿真模式，点击信封图标Add Simple PDU。
  2. 先点击PC0作为源设备，再点击PC1作为目标设备。
  3. 点击自动捕获播放按钮，直观查看数据包从PC0→交换机→PC1完整转发流程，点开数据包可查看各层封装信息。

四、实验结果

4.1 连通性测试汇总表

测试项 源设备(IP) 目标设备(IP) 测试结果 丢包率
ping连通测试 PC0(192.168.1.1) PC1(192.168.1.2) ✅ 成功 0%

4.2 成果说明

  1. 成功完成Cisco Packet Tracer安装、账号登录，软件功能可正常使用；
  2. 完成基础局域网拓扑搭建，设备线缆连接无报错；
  3. 两台终端IP配置在同一网段，双向ping测试全部收到回复，网络通信正常；
  4. 仿真模式下可完整观测数据包转发过程，理解二层交换机数据转发逻辑。

五、知识总结

三种主流网络软件架构对比

架构类型 全称 核心特征 典型例子
C/S架构 客户端/服务器 需要安装专用客户端，两端分工协作 网络游戏、微信、Packet Tracer
B/S架构 浏览器/服务器 仅浏览器即可访问，业务逻辑在服务端 淘宝、知乎、在线文档
P2P架构 点对点 每个节点同时充当客户端与服务器 BT下载、局域网传输工具

六、心得体会

技术层面

简单了解了Cisco Packet Tracer这一工具，并且感受了小型局域网的搭建，还通过了仿真测试直观看到了数据包的传输过程。

思政层面

目前国内在网络仿真工具方面存在欠缺，配套的工具较少。我们需要持续学习，助力我们国家完善网络教学生态，让信息技术自主可控。

实验四 Web站点构建

Posted on 2026-07-09 Edited on 2026-07-12 In 实验报告

一、实验目的

  1. 使用静态站点构建工具（Hexo）构建一个静态Web博客站点；
  2. 了解和掌握Web应用开发的基本要素与流程：环境准备 → 项目初始化 → 内容编写 → 主题配置 → 本地预览 → 部署上线；
  3. 理解静态站点生成器（SSG）的工作原理及其与动态站点的区别。

二、实验环境

  • 操作系统：Windows 10/11
  • 网络：需要互联网连接
  • 前置条件：已完成实验二（Node.js、npm、Git已安装并配置）
  • 所需账号：GitHub账号（用于免费部署）

三、实验过程

步骤一：环境校验，确认基础工具可用

  1. 打开CMD命令提示符，依次执行三条校验命令，验证Node、npm、Git全部正常安装。

步骤二：全局安装Hexo CLI工具

  1. 在CMD执行全局安装命令。
  2. 安装完成后输入hexo -v，查看Hexo完整版本与依赖信息。

步骤三：初始化Hexo博客项目

  1. 创建博客存放目录并进入文件夹。
  2. 执行hexo init my-blog初始化命令生成博客模板。
  3. 进入项目目录，自动安装依赖包。
  4. 查看项目完整目录结构。

步骤四：本地启动预览服务

  1. 在my-blog目录执行启动命令。
  2. 控制台输出INFO Hexo is running at http://localhost:4000即启动成功。

步骤五：新建并编辑博客文章

  1. 执行hexo new “我的第一篇博客”新建文章。
  2. VS Code打开生成的Markdown文件，修改标题、标签、正文测试内容。
  3. 保存后生成静态页面，重启服务刷新查看效果。

步骤六：安装NexT美化主题

  1. 使用Git克隆NexT主题至themes文件夹。
  2. 打开_config.yml，修改 heme: next。
  3. 清理缓存、重新生成页面并启动服务预览新主题样式。

步骤七：GitHub仓库创建与部署配置

  1. 登录GitHub，新建公开仓库（用户名.github.io）。

  1. 在本地项目安装Git部署插件。
  2. 修改_config.yml文件末尾deploy配置，填入GitHub仓库远程地址。
  3. 执行一键部署命令：hexo clean && hexo generate && hexo deploy

步骤八：访问线上博客站点

部署完成后，浏览器打开https://yj-zj.github.io，验证线上博客文章、主题样式与本地完全一致。

四、实验结果

成果说明

  1. 完整搭建Hexo静态博客开发环境，所有依赖工具正常识别调用；
  2. 完成博客项目初始化，熟练掌握Hexo全套命令；
  3. 成功替换NexT第三方主题，实现博客页面美化；
  4. 完成GitHub仓库创建、部署插件配置，将静态站点免费部署至GitHub Pages。

五、知识总结

静态站点 vs 动态站点对比

对比维度 静态站点（Hexo/Hugo） 动态站点（WordPress）
页面生成时机 本地提前预编译HTML 用户访问时后端实时渲染
数据库 无需数据库 必须搭配数据库
访问速度 极快，纯静态文件 较慢，需执行后端查询
安全风险 无后端攻击面，安全性高 存在后台、数据库漏洞风险
部署方式 仅上传静态文件 搭建服务器+数据库环境
适用场景 个人博客、技术文档站 电商、论坛、后台管理系统

Hexo核心常用命令

命令 简写 功能说明
hexo init 项目名 - 初始化博客项目模板
hexo server hexo s 启动本地预览服务
hexo new “标题” - 创建Markdown博客文章
hexo generate hexo g 编译生成静态HTML文件
hexo clean - 清理缓存与旧静态文件
hexo deploy hexo d 推送站点至远程仓库

六、出现问题

  2. 问题2：部署完成博客样式丢失 → 修改_config.yml中url字段，填写完整GitHub Pages线上地址。
